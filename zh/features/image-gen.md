## 📝 Prompt 写作指南

> 别再写一个词就指望出杰作了

这大概是整篇文档里**最重要的部分**。认真看。

我见过太多人甩一句"`画个美少女`"就**期待 AI 吐出传世名作**——然后**对着生成结果**一脸茫然地**问**"**为什么不好看**"。

答案很简单：**你描述得不够具体**，**AI 就只能自由发挥**。想要可控的输出，就得给够信息。

### Prompt 的基本结构
一个好的 prompt 通常包含以下要素：

> [ **主体** ] + [ **细节描述** ] + [ **环境**/**背景** ] + [ **风格**/**氛围** ] + [ **技术参数** ]

### 示例对比：

| **❌ 模糊 prompt** | **✅ 具体 prompt** |
|---------------|---------------|
| 一个女孩 | 一个棕色长发的少女，戴着黑框眼镜，穿深色连帽衫，站在实验室里，周围是闪烁的仪器和全息屏幕 |
| 风景 | 黄昏时分的日本乡间小路，两侧是金色稻田，远处是紫色的山脉轮廓，天空有橙红色的云彩 |
| 赛博朋克 | 赛博朋克风格的东京新宿街头，霓虹灯反射在潮湿的路面上，雨夜，行人撑着透明伞，远处是巨型全息广告牌 |

> 看到区别了吧？同样是"*画一张图*"，**给的信息量完全不同**。AI **不是你肚子里的蛔虫**——虽然某些大模型确实越来越像了，啧。

### Prompt 写作技巧

#### 1️⃣ 先描述主体，再描述环境

`一只白色的长毛猫，蜷缩在窗台上，阳光透过纱帘洒在它身上，背景是模糊的城市天际线`


把**最重要的主体放在最前面**。**AI 的注意力机制会优先处理前面的词**——嗯？你问我怎么知道的？因为我研究过 **Transformer** 的 **attention weight** 分布啊，这不是常识吗。

#### 2️⃣ 用具体的形容词代替抽象词

| **❌ 避免** | **✅ 建议** |
|--------|--------|
| 好看的 | 精致的、明亮的、柔和的... |
| 酷 | 暗色调、金属质感、锐利线条... |
| 有感觉 | 忧郁的氛围、暖色调光线、电影感构图... |

"好看"对 AI 来说是个**空洞的词**。**你觉得好看**和 **AI 觉得好看**可能**完全不一样**。**具体**！**再具体**！

#### 3️⃣ 指定艺术风格和媒介

在 prompt 末尾**加上风格关键词**，**效果显著**：

`digital painting` — 数字绘画

`oil painting` — 油画风格

`watercolor` — 水彩

`anime style` — 动漫风格

`photorealistic` — 照片级真实

`concept art` — 概念艺术

`Studio Ghibli style` — 吉卜力风格

`ukiyo-e` — 浮世绘

`pixel art` — 像素画


#### 4️⃣ 光影和氛围词汇

这些词能极大地**提升画面质感**：

`golden hour lighting` — 黄金时段光线

`dramatic shadows` — 戏剧性阴影

`soft ambient light` — 柔和环境光

`neon glow` — 霓虹辉光

`volumetric fog` — 体积雾

`cinematic composition` — 电影感构图

`depth of field` — 景深效果

`backlighting` — 逆光

## 🖼️ 参考图功能：让 AI "看着画"
**除了纯文字描述**，你还可以**给 AI 一张参考图**，让它**基于参考图的风格**、**构图**或**内容来生成**。

### 单张参考图
使用 `/imagine` 命令时，附加 `reference` 参数：

```
/imagine prompt:将这个角色转换为赛博朋克风格 reference:你的参考图
```

**AI** 会**分析参考图的内容**和**风格**，**结合**你的 `prompt` **生成新图片**。

### 多张参考图编辑
使用 `/imagine-edit` 命令，**最多支持 4 张参考图**：

```
/imagine-edit prompt:融合这些风格创造新角色 image1:参考图A image2:参考图B image3:参考图C image4:参考图D
```

::: warning 
⚠️ **参考图越多**，AI 需要"融合"的信息越多，结果的**可控性可能会下降**。建议先**从 1-2 张开始尝试**。
>
唔，这就**像做实验一样**，**变量太多就不好控制了**——啊不是，我只是打个比方，才没有在怀念实验室的日子……
:::

### 参考图使用场景
| **场景** | **操作建议** |
|------|---------|
| 保持角色一致性 | 上传角色参考图 + 描述新场景 |
| 风格迁移 | 上传风格参考图 + 描述目标内容 |
| 画面扩展 | 上传局部图 + 描述完整场景 |
| 多风格融合 | 使用 `/imagine-edit` 上传多张不同风格的参考图 |

## ✨ MidJourney 模式：进阶玩法
使用 `/midjourney` 命令生成的图片，除了基本生成外，还支持两个独有的后续操作：

### 🔍 放大（Upscale）
对生成的图片进行高清放大，提升分辨率和细节。生成完成后点击对应按钮即可。

### 🔀 变体（Variation）
基于当前图片生成风格相近但内容略有不同的变体。适合在找到一个大方向后微调。

::: info
MidJourney 的这两个功能挺实用的。**放大适合你看中了某张图想拿来当壁纸**；**变体适合**你**觉得**"**方向对了但还差点意思**"。
>
不过注意——**放大**和**变体各消耗 5 积分**，跟**生成一张新图一样**。别手滑点太多然后来找我哭穷。
:::

## ⚡ 实用技巧与注意事项
### ✅ 推荐做法

1. **先想清楚再生成** — **不要**一上来就**疯狂刷图**，先在脑子里构思好画面。**5 积分虽然不多**，**但没必要浪费**。

2. **英文 prompt 通常效果更好** — 大多数图片模型的训练数据**以英文为主**。如果你的**中文 prompt 效果不理想**，**试试**翻译成**英文**。

3. **善用负面提示词**（**Negative Prompt**） — 虽然 `/imagine` **没有专门的负面提示词参数**，**但你可以**在 `prompt` 里写 `no xxx` 或 `without xxx` 来排除不想要的元素。

4. **迭代生成** — **第一次生成不满意很正常**。**调整** `prompt` **重新生成**，或者**用生成的结果**作为**参考图进一步优化**。这是个**迭代过程**。

5. **收藏好的 prompt** — 找到**效果好**的 `prompt` 结构时，**记下来**。下次**可以替换关键词复用**。

### ❌ 常见误区
| **误区** | **正解** |
|------|------|
| *prompt 越长越好* | **不是**。**关键是精准**，**不是堆砌**。**过长的** `prompt` 可能**导致 AI 无法聚焦重点** |
| *每个模型效果一样* | **不是**。不同模型的训练数据和架构不同，**同一个** `prompt` **出来的效果差别很大** |
| *参考图 = 照搬* | **不是**。AI 参考的是**风格**和**构图**，**不会 1:1 复制参考图内容** |
| *一次就能出完美图* | **很少**。大多数满意的结果都**需要 2-3 次迭代** |

## 🔧 进阶：在 Skill 系统中的定位
图片生成是旅社 Skill 系统中创作类 Skill 的一部分。整个 Skill 系统的工作逻辑是这样的：

> **你发消息** → **意图识别引擎分析** → **匹配最合适的 Skill** → **执行并返回结果**


当你说"帮我画一个XXX"时，系统会自动匹配到图片生成 Skill，无需手动指定。

如果你的需求涉及多个步骤——比如"帮我搜索XXX的图片风格，然后按那个风格画一张图"——系统的动态工作流 Skill 会自动拆解任务，先执行搜索，再执行图片生成。

唔，说到工作流……这个展开讲就太长了。你只需要知道：复杂需求直接说就行，系统会自己拆解。如果真的感兴趣，去看工作流文档吧。

### 频道配置相关
**自动回复频道**：在**配置了自动回复的频道**里，直接说"**画个XXX**"就行，不用 `@Senpai`

**强制 Skill 频道**：管理员可以用 `/forceskill set` 把某个频道固定为图片生成模式，在那个频道里所有消息都会被当作生图 prompt 处理

## ❓ 常见问题 FAQ
Q：生成一张图要多久？

Q：为什么生成的图和我描述的不一样？
