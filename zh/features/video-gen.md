## ⚙️ 参数说明

`/video` 支持以下参数：

| **参数** | **说明** | **是否必填** |
|-----|------|---------|
| `prompt` | **视频内容的文字描述** | ✅ 必填 |
| `model` | 选择视频**生成模型** | 可选 |
| `image` | 提供**参考图片**（图生视频） | 可选 |
| `video` | 提供**参考视频**（视频转视频） | 可选 |
| `negative_prompt` | **负面提示词**，描述你**不想要**的内容 (**仅Wan 2.6 支持**) | 可选 |
| `last_image` | **尾帧图片** (与首帧一起使用,仅部分模型支持) | 可选 |
| `keep_sound` | **是否保留参考视频的原声** (**仅Kling模型的视频转视频支持**,默认保留)  | 可选 | 
| `audio` | **背景音乐URL** (**仅Wan 2.6 支持**) | 可选 | 
| `shot_type` | **镜头类型** (**仅 Wan 2.6 支持**) | 可选 |

有全部参数的模板（可直接复制粘贴）：
```
/video prompt: model: image: video: last_image: keep_sound: negative_prompt: audio: shot_type:
```

## 🖼️ 三种生成模式

### 模式一：文生视频

最常用的方式。用**文字**描述**你想要的画面**，AI 凭空生成。

```
/video prompt:一只猫咪在阳光下打盹，慢镜头，温暖的光线
```

### 模式二：图生视频

上传一张**参考图片**，AI 让它"**动起来**"。

```
/video prompt:让图片中的场景动起来，风吹过树叶 image:（上传图片）
```

> 图生视频的优点是能保留你图片的构图和风格，生成结果和参考图有明显的关联性。

### 模式三：视频转视频

上传一段**参考视频**，AI 在其**基础上**进行 *风格转换* 或 *内容改造* 。

```
/video prompt:将视频转换为水墨画风格 video:（上传视频）
```

> 唔……这个模式**支持的模型有限**，目前 **Kling**、**Wan 2.6**、**Seedance 2.0** 支持参考生成。**选模型的时候**注意一下。

## 💡 如何写出好的视频 Prompt

### 基础结构

```
[主体] + [动作/状态] + [环境/背景] + [镜头语言] + [风格/氛围]
```

### 示例对比

❌ **太模糊**
```
一个很好看的视频
```

✅ **清晰具体**
```
一位女孩站在樱花树下，微风吹过，花瓣缓缓飘落，近景慢镜头，暖色调，电影质感
```

### 实用关键词参考

**镜头语言**：

`close-up（近景）` `wide shot（广角）` `aerial view（航拍）` `slow motion（慢镜头）` `time-lapse（延时摄影）` `tracking shot（跟随镜头）`

**画面风格**：

`cinematic（电影质感）` `photorealistic（写实）` `anime style（动漫风格）` `watercolor（水彩）` `8K quality`

**光线与氛围**：

`golden hour（黄金时刻）` `neon lights（霓虹灯）` `dramatic lighting（戏剧光线）` `misty（薄雾）` `rainy atmosphere（雨天氛围）`

**运动描述**：

`flowing（流动）` `gentle breeze（微风）` `waves crashing（海浪拍打）` `falling petals（花瓣飘落）`

### 负面提示词的使用

`negative_prompt` 参数可以排除你不想要的内容：

```
negative_prompt: blurry, low quality, distorted, text, watermark
```

> 这个参数很有用，加上之后生成质量明显提升——嘛，顺便告诉你。

## ⏳ 生成流程说明

视频生成是**异步处理**的，和图片不同，它**不会立刻出结果**。

1. 提交请求后，Bot 返回确认消息和「**查看状态**」按钮
2. 点击按钮随时**查看生成进度**
3. 生成完成后可以直接在 Discord 内**预览和下载**

**大概等多久？**

取决于**模型**和**当前队列情况**，通常在 **3–10 分钟**之间。旗舰**模型**可能需要更长时间。

> 等待的时候去做点别的吧……比如喝杯 Dr Pepper 什么的。这是经验之谈。

## 💡 进阶技巧

### 技巧一：先用经济型测试 Prompt

第一次尝试一个新概念时，用 *Wan 2.2 Plus* （*50 积分*） 测试效果，满意后**再换旗舰模型出高质量版本**。节省积分，也减少等待时间。

### 技巧二：图生视频比文生视频更可控

如果你**对最终画面有明确要求**，先用**图片生成功能** `/imagine` 生成一张**满意的静态图**，再用 `/video` 的 `image` 参数让它动起来。**这样构图和风格会更符合预期**。

::: tip 更多图片生成指南

**欲知更多生成图片的小技🡖巧🡕**？

👉 [图片生成指南 → Promot 写作指南](/zh/features/image-gen#📝-prompt-写作指南)

:::

### 技巧三：善用 negative_prompt

视频里容易出现的问题—— *模糊* 、 *扭曲的人脸* 、 *莫名的文字* ——都可以放进 `negative_prompt` 提前排除：

```
negative_prompt: blurry, distorted face, text, watermark, low quality, flickering
```

### 技巧四：搭配其他功能使用

| **搭配** | **效果** |
|-----|-----|
| 先 `/imagine` 生成图 → 再 `/video` 图生视频 | 精确控制画面风格和构图 |
| 先 `/music` 生成音乐 → 手动合并视频 | 为视频搭配专属 BGM |

## ❓ 常见问题

### **Q：生成的视频质量不满意怎么办？**
A：调整 Prompt 描述，加入更具体的镜头语言和风格词，并在 `negative_prompt` 中排除不想要的元素，然后重新生成。同样的 Prompt 每次结果都有差异，多试几次是正常的。

### **Q：哪个模型最好？**
A：没有绝对的"最好"，取决于需求。写实场景推荐 Sora 系列；人物动作推荐 Kling；性价比首选 Wan 系列；中文内容或自然风景推荐 Seedance。

### **Q：图生视频支持哪些图片格式？**
A：常见格式（PNG、JPG）都支持，直接上传附件即可。

### **Q：视频可以生成多长？**
A：**取决于模型**，通常在 **4–10 秒之间**。大多数模型**默认生成约 5 秒的短视频**。

### **Q：积分不够怎么办？**
A：*每日签到* + *聊天任务*可以稳定回血。

### **Q：可以生成有声音的视频吗？**
A：目前视频生成功能主要输出画面。如需配音，可以单独使用 `/voice` 生成语音或 `/music` 生成背景音乐，再自行合并。
